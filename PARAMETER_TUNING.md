# 参数调优指南

## 🎯 目标

通过系统的参数调优，使小车能够：n1. ✅ 稳定跟踪黑线
2. ✅ 弯道不丢线
3. ✅ 快速响应方向变化
4. ✅ 减少或消除振荡

## 📊 PID参数详解

### 1. Kp（比例系数）- 控制响应强度

**作用**：直接控制对偏差的响应程度

| Kp值 | 行为 | 适用场景 |
|------|------|----------|
| 太小 (<1.0) | 转向不足，跟不上线 | - |
| 适中 (1.5-2.5) | 快速响应，不易振荡 | ✅ 推荐 |
| 太大 (>3.0) | 转向过度，容易振荡 | 弯道多的赛道 |

**调整方法**：
```
初始值：Kp = 1.5
↓
小车跟踪效果不好？Kp += 0.2
↓
小车开始振荡？Kp -= 0.1
↓
找到最大的Kp值而不振荡
```

### 2. Kd（微分系数）- 阻尼作用

**作用**：预测偏差的变化趋势，提供阻尼

| Kd值 | 行为 | 适用场景 |
|------|------|----------|
| 太小 (<1.0) | 不足以防止超调 | - |
| 适中 (2.5-3.5) | 平滑运动，很少超调 | ✅ 推荐 |
| 太大 (>4.0) | 响应变慢，转向不足 | - |

**调整方法**：
```
初始值：Kd = 2.0
↓
发现小车经过弯道时振荡？Kd += 0.2
↓
转向变慢了？Kd -= 0.1
↓
找到平衡点
```

### 3. Ki（积分系数）- 消除静态误差

**作用**：消除系统的持久偏差

| Ki值 | 行为 | 适用场景 |
|------|------|----------|
| 0 | 有持久偏差 | - |
| 小 (0.05-0.15) | 消除偏差，稳定 | ✅ 推荐 |
| 太大 (>0.3) | 积分饱和，易振荡 | - |

**调整方法**：
```
一般情况：Ki = 0.1（保持不变）

仅在以下情况调整：
- 小车持续偏向一侧 → Ki += 0.02
- 小车容易振荡 → Ki -= 0.02
```

## 🚀 完整调优流程

### 第一步：基础速度调优

1. 将小车放在直线黑线上
2. 设置 `st_speed = 2000`（低速）
3. 按下K5启动巡线
4. 观察小车是否能稳定跟踪
5. 逐步增加 `st_speed`（+500），观察最大能跟踪的速度

**推荐基础速度范围**：
```
简单赛道（宽直线）：3500-4000
中等复杂度：3000-3500
高难度（多弯道）：2500-3000
```

### 第二步：Kp调优（最重要）

1. 固定 `st_speed = 3000`
2. 从 `kp = 1.0` 开始
3. 用K1（Kp+）逐步增加Kp
4. 观察转向响应：
   - 转向太慢？继续增加Kp
   - 开始振荡？停止，记录当前Kp
5. 返回Kp值-0.1作为最终值

**检查清单**：
- [ ] 小车能快速跟踪直线黑线
- [ ] 转向响应及时
- [ ] 没有过度振荡

### 第三步：Kd调优（重点是消除振荡）

1. 固定Kp在第二步得到的最优值
2. 从 `kd = 2.0` 开始
3. 让小车通过90度弯道
4. 用K3（Kd+）逐步增加Kd
5. 观察弯道处理：
   - 经过弯道时振荡？增加Kd
   - 转向反应变慢？减少Kd
6. 找到平衡点

**检查清单**：
- [ ] 弯道处平滑无振荡
- [ ] 转向不过度（无超调）
- [ ] 直线段稳定

### 第四步：Ki微调（通常不需要改）

1. 在较长的直线段测试
2. 观察小车是否持续偏向一侧
3. 如果有偏差，微调Ki（±0.02）
4. 一般情况保持Ki = 0.1

## 📋 不同赛道的推荐参数


## 🔍 调试技巧

### 1. OLED实时监控

**五路传感器显示**
```
LL LM MM RM RR
20 35 85 40 15

说明：中间传感器值最高，说明小车在黑线中央
```

**误差值含义**
```
Err: 5.0  → 偏向右侧，需要左转
Err: 0.0  → 完美在线
Err: -5.0 → 偏向左侧，需要右转
```

### 2. 串口输出（可选）

在 `pid_line_tracking()` 函数中添加：
```c
printf("Err:%5.1f, P:%6.0f, I:%6.0f, D:%6.0f\r\n",
       error[0], p_term, i_term, d_term);
```

这样可以在串口工具中看到实时的PID各项值。

### 3. RGB LED指示

- **黄色** - 校准中
- **绿色** - 正常巡线
- **红色** - 丢线警告
- **蓝色** - 弯道通过

## ⚠️ 常见问题排查

### Q1: 小车不能很好地跟踪黑线

**可能原因**：
1. 传感器未校准
2. Kp太小
3. 黑线宽度不合适

**解决方案**：
```
优先级 1：按K6重新校准传感器
优先级 2：按K1增加Kp（+0.1，多次）
优先级 3：检查黑线宽度（应为3-5cm）
```

### Q2: 小车在巡线时振荡

**可能原因**：
1. Kp或Kd太大
2. st_speed太高
3. 传感器干扰

**解决方案**：
```
优先级 1：按K2减少Kp（-0.1，多次）
优先级 2：按K4减少Kd（-0.1，多次）
优先级 3：代码中降低st_speed重新编译
```

### Q3: 转向反应太慢

**可能原因**：
1. Kp太小
2. st_speed太低
3. 传感器灵敏度不足

**解决方案**：
```
优先级 1：按K1增加Kp（+0.1，多次）
优先级 2：代码中提高st_speed重新编译
优先级 3：重新校准传感器（K6）
```

### Q4: 弯道容易丢线

**可能原因**：
1. 速度太快
2. Kd不足
3. 弯道半径过小

**解决方案**：
```
优先级 1：代码中降低st_speed重新编译
优先级 2：按K3增加Kd（+0.1，多次）
优先级 3：检查赛道设计（弯道是否过急）
```

### Q5: 直线段持续偏向一侧

**可能原因**：
1. 传感器校准不准确
2. 左右电机功率不匹配
3. 小车底盘不水平

**解决方案**：
```
优先级 1：按K6重新校准传感器
优先级 2：微调Ki（+-0.02）
优先级 3：检查小车硬件（轮子、底盘）
```

## 💡 高级技巧

### 1. 速度与参数的关系

```
速度越高：
- 需要更大的Kp以保证快速响应
- 需要更大的Kd以防止过度转向
- 容易产生振荡

速度越低：
- 可以使用较小的Kp
- Kd可以适度降低
- 转向可以更激进
```

### 2. 黑线宽度的影响

```
黑线很宽（>5cm）：
- Kp可以较小（2.0以下）
- 容易找到黑线

黑线标准（3-5cm）：
- Kp = 2.0（推荐）

黑线很窄（<3cm）：
- Kp需要较大（2.5-3.0）
- 需要更好的Kd
```

### 3. 编码器反馈（进阶）

如果集成编码器反馈，可以实现：
- 精确的速度控制
- 直线自适应调速
- 弯道防滑检测

## 📚 参考资源

### PID控制原理
```
输出 = Kp×e(t) + Ki×∫e(τ)dτ + Kd×de(t)/dt

P项：快速响应，直接作用
I项：消除误差，但有延迟
D项：预测趋势，提供稳定性
```

### 调优黄金法则
1. **一次调一个参数** - 确保能看到该参数的影响
2. **记录每次调整** - 便于对比和回退
3. **小幅度调整** - 避免过度修正
4. **充分测试** - 多个赛段确认效果
5. **保存最优值** - 不要遗忘好的参数
