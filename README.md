# GD32F303 电磁智能巡线小车系统

## 📋 项目概述

基于 GD32F303RCT6 单片机的完整电磁巡线系统，采用PID控制算法实现高精度黑线跟踪。

### 🎯 核心特性

- ✅ **5路电磁传感器** - 高精度线位检测
- ✅ **PID控制算法** - 自适应参数调优
- ✅ **编码器反馈** - 精确距离控制
- ✅ **OLED实时显示** - 调试参数监测
- ✅ **RGB LED指示** - 状态提示
- ✅ **按键调试** - 6键参数调整
- ✅ **丢线保护** - 应急处理机制
- ✅ **传感器校准** - 自动最大值采样

## 🚀 快速开始

### 1. 系统初始化
```c
void main(void) {
    car_init();                           // 初始化所有硬件
    current_mode = MODE_CALIBRATION;      // 进入校准模式
    
    while(1) {
        xunxian_process();                // 主巡线处理
        tune_parameters();                // 按键参数调试
        delay_1ms(10);
    }
}
```

### 2. 校准流程
1. 启动系统，OLED显示 "Calibration..."
2. 将小车放在黑线上
3. 系统自动采样2000次获取最大值
4. 校准完成后自动进入巡线模式

### 3. 巡线运行
- 自动读取5路传感器
- 计算实时偏差值
- PID控制电机速度
- OLED实时显示参数

## 📊 PID参数说明

| 参数 | 推荐值 | 调整影响 |
|------|--------|----------|
| **Kp** | 2.0 | 控制响应速度，值越大越快 |
| **Ki** | 0.1 | 消除静态误差 |
| **Kd** | 3.0 | 阻尼作用，防止超调 |
| **st_speed** | 3000 | 基础速度（PWM值0-10000） |

### 快速调优流程
1. **先调Kp** - 从1.5开始，逐步增加至合适值
2. **再调Kd** - 增加Kd防止震荡
3. **最后调Ki** - 消除持久偏差

详见 `PARAMETER_TUNING.md` 获取详细调优指南。

## 🎮 按键功能

| 按键 | 功能 | 说明 |
|------|------|------|
| **K1** | Kp+ | 增加比例系数 |
| **K2** | Kp- | 减少比例系数 |
| **K3** | Kd+ | 增加微分系数 |
| **K4** | Kd- | 减少微分系数 |
| **K5** | 启停 | 切换巡线/停止 |
| **K6** | 重新校准 | 重新进行传感器校准 |

## 📺 OLED显示内容

```
第一行：LL LM MM RM RR      (五路传感器值 0-100%)
第二行：Err: XX.X          (实时偏差值)
第三行：Mode: TRACK        (运行模式)
第四行：Kp: 2.0 Kd: 3.0   (当前参数)
```

## 🔍 调试技巧

### 传感器值监控
- OLED显示五路传感器的实时归一化值（0-100%）
- 可以直观看到小车相对黑线的位置

### 误差监控
- 误差值接近0 = 小车在黑线中央
- 误差值>0 = 偏向右侧，需要左转
- 误差值<0 = 偏向左侧，需要右转

### 调试流程
```
1. 观察基础速度下的巡线效果
2. 检查转向是否及时
3. 调整Kp使转向更敏感
4. 调整Kd消除过度转向
5. 在复杂赛道测试稳定性
```

## ⚠️ 常见问题

### Q: 小车不能很好地跟踪黑线
**A:** 重新校准传感器（K6），确保黑线宽度3-5cm

### Q: 小车在巡线时振荡
**A:** 减少Kp或增加Kd，降低基础速度

### Q: 转向反应太慢
**A:** 增加Kp值，提高基础速度

### Q: 弯道容易丢线
**A:** 降低st_speed，增加Kd，确保传感器检测范围内

更多问题见 `PARAMETER_TUNING.md` 的"常见问题"章节。

## 🎓 学习资源

### PID控制原理
```
输出 = Kp×误差 + Ki×∫误差 + Kd×d(误差)/dt

- P项：快速响应，直接作用于误差
- I项：消除静态误差，需要延迟
- D项：预测趋势，提供阻尼
```

### 误差计算
```
error = (rm - lm) / (rm + lm) × 100

优先级：中间传感器 > 两侧传感器 > 最外传感器
```
